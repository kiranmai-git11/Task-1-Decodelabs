Introduction to IoT &
Device Communication

Understanding how billions of connected devices sense, communicate, and act in the physical world.

IoT Architecture Layers
LAYER 01

📡 Perception / Device Layer

Physical devices and sensors that interact with the real world. They collect raw data like temperature, motion, light, humidity, and location.

Sensors Actuators Microcontrollers Arduino Raspberry Pi ESP32
LAYER 02

🔗 Network / Connectivity Layer

Handles data transmission between devices and backend systems. Uses various protocols depending on range, power, and bandwidth needs.

Wi-Fi Bluetooth Zigbee LoRaWAN MQTT CoAP
LAYER 03

☁️ Application / Cloud Layer

Processes, stores, analyzes, and visualizes data. Hosts dashboards, automation rules, machine learning models, and APIs.

AWS IoT Azure IoT Hub Google Cloud ThingSpeak Node-RED
Data Flow in an IoT System
🌡️

1. Sense

Sensor reads physical data (temp, humidity, motion)

⚙️

2. Process

Microcontroller converts raw signals to digital data

📶

3. Transmit

Data sent via Wi-Fi / MQTT to broker or gateway

☁️

4. Store & Analyze

Cloud server receives, stores, and processes data

📊

5. Act & Visualize

Dashboard displays results; automation triggers actions

Real-World IoT Use Cases
🏠

Smart Home

Automated lighting, thermostats, security cameras, and voice assistants connected via Wi-Fi.

🏥

Healthcare

Wearable monitors tracking heart rate, blood oxygen, and glucose levels in real-time.

🌾

Smart Agriculture

Soil moisture sensors and weather stations enabling precision irrigation and crop monitoring.

🏭

Industrial IoT

Predictive maintenance by monitoring vibration, heat, and pressure in factory machinery.

🚗

Smart Transport

GPS tracking, fleet management, and autonomous vehicle sensors sharing road data.

🏙️

Smart City

Intelligent traffic lights, waste level sensors, and air quality monitoring networks.

Communication Protocols Comparison
Protocol Range Power Usage Data Rate Best For
Wi-Fi (802.11) ~100m High Up to 1 Gbps Home automation, cameras
Bluetooth / BLE ~10–50m Very Low 1–3 Mbps Wearables, health devices
Zigbee ~10–100m Low 250 Kbps Smart lighting, mesh networks
LoRaWAN ~2–15 km Very Low 0.3–50 Kbps Agriculture, smart city
MQTT (over TCP) Internet Low–Med Varies Cloud messaging, telemetry
4G / 5G Nationwide High Up to 10 Gbps Vehicles, remote monitoring

🔑 Key Takeaway

IoT connects the physical and digital worlds by using sensors to collect data, networks to transmit it, and cloud platforms to store, analyze, and act on it. Choosing the right protocol and architecture depends on power constraints, range, data volume, and the specific application domain.
